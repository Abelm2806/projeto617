Como trabalhamos

A nossa equipa organiza o trabalho através de branches para que cada elemento possa desenvolver a sua parte do projeto sem interferir com o trabalho dos colegas.

A revisão das alterações é importante para encontrar erros e melhorar o código.

Regras de Trabalho:

  • Fazer commits pequenos e frequentes.
  • Testar as alterações antes de enviar para o GitHub.
  • Rever o trabalho dos colegas através de Pull Requests.

Objetivo do projeto

Ao longo deste projeto, a equipa vai praticar a utilização de Git e GitHub para colaborar de forma organizada no desenvolvimento de páginas web.

O Git e o GitHub facilitam a colaboração em equipa, permitindo organizar o trabalho, acompanhar alterações e desenvolver projetos de forma mais eficiente.
